# Техническое задание: Мессенджер «Lunar»

## 1. Суть проекта
**Lunar** — это минималистичный веб-мессенджер в темных тонах. Главный упор я сделал на компактность интерфейса и работу в реальном времени. Сайт должен быть легким, без громоздких элементов и лишних отступов.

## 2. Функциональные требования

### 2.1. Аккаунты и сессии
* **Регистрация и вход:** По никнейму (ID) и паролю. Пароли хешируются.
* **Профиль:** В шапке отображается ник текущего юзера и кнопка «Выйти» (Logout), которая уничтожает сессию и редиректит на вход.
* **Аватарки:** Чтобы не перегружать проект работой с файлами, аватарка генерируется автоматически — это кружок с первой буквой имени пользователя. Потом в будущем, возможно добавлю возможность ставить свою аватарку.

### 2.2. Система чатов и поиск
* **Поиск:** Поиск собеседника по точному никнейму через правую панель.
* **Список чатов (слева):** Формируется динамически на основе последних сообщений. 
* **Индикация:** Голубой кружок с количеством непрочитанных сообщений. При открытии чата счетчик сбрасывается.

### 2.3. Механика диалога
* **Визуал:** Сообщения «лесенкой» (мои — справа, чужие — слева). Текст компактный, без жирных обводок.
* **Разделители дат:** Центрированный текст. Логика: «Сегодня», «Вчера», далее формат «ДД месяц» (например, «8 марта»).
* **Время:** У каждого сообщения отображается время отправки (МСК).

## 3. Техническая реализация

### 3.1. Стек и БД
* **Стек:** Python (Flask) + Flask-SocketIO + SQLAlchemy (ORM).
* **БД (SQLite):** 
    * `User` (id, username, display_name, password_hash)
    * `Message` (id, sender_id, receiver_id, text, timestamp, is_read)
    * *Примечание:* Список чатов формируется на лету через запрос к таблице сообщений (для упрощения архитектуры).

### 3.2. WebSocket события (Real-time)

| Событие | Направление | Описание |
| :--- | :--- | :--- |
| `send_message` | Клиент -> Сервер | Отправка текста и ID получателя |
| `new_message` | Сервер -> Клиент | Мгновенная доставка сообщения во фронтенд |
| `mark_read` | Клиент -> Сервер | Сигнал базе данных, что сообщения в чате прочитаны |

### 3.3. Качество проекта (по критериям)
* Верстка на **Flexbox/Grid** для исключения пересечения элементов.
* Наличие файла `requirements.txt` со всеми зависимостями.
* Код структурирован по модулям: отдельно логика БД, отдельно обработка сокетов.

## 4. Дизайн (Lunar Style)
* Фон: Глубокий темный (#0B0E11).
* Текст: Светло-серый и белый.
* Акцент: Голубой (#00AFFF) для уведомлений и важных элементов.
