# Техническое задание: Мессенджер «Lunar»

### Основные функциональные требования

**1) Авторизация и профили**
* Регистрация через уникальный никнейм, отображаемое имя и пароль.
* Вход по нику и паролю. Пароли хранятся в хешированном виде (pbkdf2:sha256).
* Функция выхода из системы с возвратом на страницу авторизации.
* Автоматическое создание аватарки (первая буква имени) с возможностью загрузки своего изображения.

**2) Список диалогов и поиск**
* Боковая панель со списком активных чатов, отсортированных по дате свежего сообщения.
* Живой поиск пользователей по точному совпадению никнейма.
* Индикация непрочитанных сообщений (синий кружок с числом).
* Автоматический сброс счетчика непрочитанных при открытии чата.

**3) Переписка в реальном времени**
* Реализация мгновенной доставки сообщений через веб-сокеты без обновления страницы.
* Визуальное разграничение: сообщения пользователя — справа (голубые), собеседника — слева (серые).
* Отображение времени отправки (МСК) и разделителей дат (Сегодня, Вчера).
* Возможность вернуть последнее сообщение в поле ввода по нажатию стрелки "Вверх".

**4) Файлы и вложения**
* Возможность отправки файлов любого типа через кнопку прикрепления.
* Отображение файлов в чате в виде кликабельных ссылок для скачивания.
* Механизм загрузки и смены аватара профиля через серверную папку static/uploads.

### Архитектура и дизайн

**Основные модули**
* **Главное окно (chat.html):** Основной интерфейс, включающий список контактов и активную область переписки.
* **Окно авторизации (auth.html):** Модуль с формами входа и регистрации.
* **Серверная логика (app.py):** Управление сессиями, маршрутизация, обработка WebSocket-событий.
* **Модуль базы данных:** Слой взаимодействия с SQLite через SQLAlchemy (таблицы User и Message).
* **Стилевой модуль (style.css):** Централизованное управление внешним видом и темами.

**Дизайн и интерфейс**
* **Тема:** Тёмная (фон #0b0e11, панели #15191c, акцент #00afff).
* **Шрифт:** Системный без засечек (Segoe UI, Arial).
* **Стиль:** Минимализм, отсутствие теней, тонкие разделители (1px), скругления 8-12px.
* **Скроллбары:** Тонкие (4px), минимально заметные в интерфейсе.

**Горячие клавиши**
* **Enter:** Отправить сообщение.
* **Стрелка вверх:** Вызвать последнее отправленное сообщение (в пустом поле).
* **Esc:** Убрать фокус с поля ввода.

**Технические детали**
* **Секретный ключ:** Lunar_Yandex_Lyceum_2026.
* **Хранение данных:** База данных SQLite в папке instance/, создаваемая автоматически.
* **Сервер:** Запуск через Flask-SocketIO (поддержка HTTP и WS на одном порту).

# Пояснительная записка к проекту «Lunar»

### 1. Что где лежит (структура проекта)

*   **app.py** — Главный файл. Здесь весь «мозг» проекта: настройки сервера, секретный ключ, работа с базой данных и логика чата через сокеты.
*   **requirements.txt** — Список модулей, которые нужно установить, чтобы всё заработало.
*   **static/** — Папка со статикой:
    *   `css/style.css` — Внешний вид: тёмная тема, синие акценты и оформление сообщений.
    *   `uploads/` — Сюда сохраняются аватарки и файлы, которые кидают в чат.
*   **templates/** — Скелет страниц:
    *   `auth.html` — Окно входа и регистрации.
    *   `chat.html` — Сам чат (список контактов и окно переписки).
*   **instance/lunar.db** — Файл базы данных. Появляется сам после запуска. В нем хранятся пользователи и вся история переписки.

---

### 2. Инструкция по запуску

Для запуска проекта на компьютере сделай три шага:

**Шаг 1: Установи модули**
Открой терминал (консоль) в папке проекта и выполни:
```bash
pip install -r requirements.txt
```

**Шаг 2: Запусти сервер**
В той же консоли вбей команду:
```bash
python app.py
```
Когда увидишь строчку `Running on http://127.0.0.1:5000`, значит всё ок.

**Шаг 3: Открой чат**
Перейди в браузере по адресу `http://127.0.0.1:5000`. 

**Как проверить работу:**
Зарегистрируйся, затем открой это же окно в **режиме инкогнито**, создай второго пользователя и найди первого через поиск по нику. Теперь можно переписываться.

---

### Полезные советы:
*   **Выключить сервер:** Нажми `Ctrl + C` в консоли.
*   **Если не работает порт:** Если адрес не открывается, поменяй в конце `app.py` порт 5000 на 5001.
*   **Файлы:** Если файлы не грузятся, просто создай папку `uploads` внутри `static` вручную.
