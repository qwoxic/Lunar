# Техническое задание: Мессенджер «Lunar»

## Основные функциональные требования

### 1) Авторизация и профили

Регистрация через никнейм, отображаемое имя и пароль. Никнейм уникальный, по нему ищут других людей. Пароли хранятся в хешированном виде.

Вход по нику и паролю. После входа пользователь попадает в главное окно чата.

Кнопка "Выйти" разлогинивает и кидает на страницу входа.

Аватарка — автоматически первая буква отображаемого имени в кружке. Можно загрузить свою картинку, она заменяет букву.

### 2) Список диалогов и поиск

Слева боковая панель со всеми пользователями, с которыми когда-либо было общение. Сортируются по дате последнего сообщения (свежие сверху).

Вверху панели строка поиска по нику. Вводишь никнейм человека, если такой есть — открывается чат с ним. Если чата раньше не было, он создаётся после первого отправленного сообщения.

Рядом с каждым диалогом — синий кружок с цифрой (сколько непрочитанных сообщений). Когда открываешь чат, все сообщения от этого собеседника становятся прочитанными.

### 3) Переписка в реальном времени

Сообщения приходят без перезагрузки страницы через веб-сокеты.

Мои сообщения — справа, голубые. Сообщения собеседника — слева, тёмно-серые.

Текст компактный, без жирных рамок и больших отступов внутри пузыря.

Рядом с каждым сообщением в самом пузыре мелким шрифтом время отправки (часы:минуты, по московскому времени).

Между сообщениями — разделители с датой: "Сегодня", "Вчера", "5 мая" и т.д. Разделитель по центру, бледный, не бросается в глаза.

Отправка сообщения по Enter. Стрелка вверх возвращает в поле ввода последнее отправленное сообщение (удобно, если надо отредактировать).

### 4) Файлы и аватары

Кнопка "+" рядом с полем ввода — прикрепление файла. Файл улетает собеседнику, отображается как ссылка на скачивание.

Смена аватара — кнопка внизу боковой панели, выбираешь картинку с компьютера, она сохраняется на сервер.

## Архитектура и дизайн

### Основные модули

**Главное окно (chat.html):** Две колонки. Слева — список чатов и панель пользователя. Справа — шапка с именем собеседника, поле переписки, снизу форма отправки сообщений.

**Серверная логика (app.py):** Все маршруты для авторизации, отправки сообщений, загрузки файлов. Обработчики сокетов для соединения и доставки сообщений.

**База данных:** SQLite + SQLAlchemy. Таблица User (id, никнейм, отображаемое имя, хэш пароля, путь к аватару). Таблица Message (отправитель, получатель, текст, путь к файлу, прочитано ли, время отправки).

### Структура страниц

**Страница авторизации (auth.html):** Две формы — вход и регистрация. Переключаются ссылкой внизу. У регистрации три поля (никнейм, имя в чате, пароль), у входа два (никнейм, пароль). Никаких капч и лишнего текста.

**Главный экран чата (chat.html):** Если выбран какой-то диалог — показываются сообщения и поле ввода. Если не выбран — в правой части просто надпись "Выберите чат".

### Горячие клавиши

**Enter** — отправить сообщение
**Стрелка вверх** (при пустом поле) — последнее отправленное сообщение
**Esc** — убрать фокус с поля ввода

### Дизайн

Тёмная тема. Фон основной #0b0e11, фоновые панели и шапки #15191c. Акцентный цвет #00afff (для моих сообщений, иконок и непрочитанных кружков).

Шрифт — системный, без засечек (Segoe UI, Arial, стандартный во всех ОС).

Скругления везде небольшие (8-12px), без теней и градиентов.

Все элементы без жирных обводок, только тонкие разделители (1px, половинная прозрачность).

Скроллбары очень тонкие (4px), незаметные.

## Технические детали

Пароли хешируются pbkdf2:sha256 через werkzeug.

Секретный ключ приложения: Lunar_Yandex_Lyceum_2026

Все статические файлы (css, загруженные картинки) лежат в папке static/.

При первом запуске база данных создаётся автоматически.

Сокеты работают на той же порту, что и http (флеск поднимает их сам через socketio.run).

#Пояснительная записка к проекту Lunar

Структура репозитория:

Lunar/
├── app.py                # Главный файл приложения (серверная логика)
├── requirements.txt      # Список зависимостей Python
├── static/               # Статические файлы
│   ├── css/
│   │   └── style.css     # Стили интерфейса (Тёмная тема)
│   └── uploads/          # Папка для пользовательских файлов
├── templates/            # HTML-шаблоны
│   ├── auth.html         # Вход и регистрация
│   └── chat.html         # Основной интерфейс мессенджера
└── instance/
    └── lunar.db          # База данных SQLite (создаётся автоматически)

Описание файлов и папок:

app.py — Главный файл приложения. Содержит настройки Flask (секретный ключ Lunar_Yandex_Lyceum_2026, путь к базе данных, папка для загрузок), модели базы данных (User, Message), маршруты для страниц и API-эндпоинты. Здесь же реализована работа с WebSocket для мгновенной отправки сообщений.

requirements.txt — Список всех Python-библиотек для работы проекта: Flask, Flask-SQLAlchemy (БД), Flask-SocketIO (веб-сокеты), Werkzeug (хеширование паролей), eventlet (асинхронный сервер) и pytz (московское время).

Папка static — Содержит статические файлы. style.css отвечает за визуальное оформление: тёмная тема (цвета #0b0e11 и #15191c), акцентный голубой цвет (#00afff), верстка сайдбара и чата, стили сообщений и аватарок. Папка uploads предназначена для хранения загруженных пользователями файлов и изображений профиля.

Папка templates — Содержит HTML-шаблоны. auth.html реализует формы входа и регистрации. chat.html включает в себя основной интерфейс мессенджера и JavaScript-код для работы с WebSocket (прием и передача сообщений в реальном времени).

Папка instance — Создается автоматически. Содержит файл lunar.db (SQLite) с двумя таблицами: user (данные пользователей) и message (история переписки, пути к файлам и временные метки).

Инструкция по запуску:

1. Установите Python версии 3.8 или выше. Проверить версию можно командой: python --version.
2. Скачайте все файлы проекта в одну рабочую директорию.
3. Откройте терминал в папке проекта и установите зависимости: 
   pip install -r requirements.txt
4. Запустите сервер командой: 
   python app.py
5. Откройте браузер и перейдите по адресу: http://127.0.0.1:5000.

Порядок работы:

1. Регистрация: Перейдите в раздел Создать аккаунт, введите уникальный никнейм и пароль.
2. Поиск: Чтобы начать чат, введите никнейм другого пользователя в строку поиска и нажмите Enter.
3. Сообщения: В открывшемся чате можно писать сообщения и прикреплять файлы через соответствующую кнопку.
4. Тестирование: Для проверки работы в реальном времени откройте приложение во втором окне браузера (в режиме инкогнито) под другим аккаунтом.

Возможные проблемы:

— Если библиотека не найдена (module not found), повторно выполните установку зависимостей через pip.
— Если порт 5000 занят, измените номер порта в последней строке файла app.py.
— Если не работают загрузки, убедитесь в наличии папки static/uploads.
— Для остановки сервера используйте комбинацию клавиш Ctrl+C в терминале.
